ARCHITECTURE OF OPERATING SYSTEM:


Operating system architecture explains the overall design of system peripherals and software components.It also explains its operations and their effectiveness as a whole


Architecture of a system comprises of various layers, where each layer has a specific well-defined task to perform

The layered architecture is broadly divided into 4 levels:

  • Level 1: Hardware
  • Level 2: Kernel
  • Level 3: Shell
  • Level 4: User Interface

Level 1- Hardware:

The physical components present in the system such as: main memory(RAM), CPU and other computer peripherals involved to perform certain tasks, makes up the hardware section of the Operating system.


Level 2- Kernel:

  • Kernel is an interface between Shell and Hardware.
  • Kernel mainly interacts with hardware components and it often referred as the heart of operating system or core part of OS.
    • Main task of Kernel:

    • Kernel receives commands from the shell (in machine level code) and it executes them with the help of system peripherals
    • While booting up the system, the system executes certain instructions (such as loading up the driver files of peripherals) which are available in Bootstrap program, which is present in Kernel. Kernel executes this bootstrap program and also other programs which are used while operating the system.
    • Kernel is also responsible for:
      • Process management
      • Input/Output device management
      • Memory management
      • File Management etc

Level 3- Shell: (Also referred as Command Line Interpreter)

Shell acts as an interface between application software/programs (user interface) and Kernel.

The main task of the Shell is:
  • Shell acts as a Command line interpreter, it understands and interpretes every line of each command received from the user.
  • Whenever we assign a command to the system, the Shell checks whether the command is valid or not (i.e correct or incorrect). If the command is incorrect, the Shell will notify the user.
  • If the command is valid, the shell converts the high level code into machine level codeand submits it to Kernel
  • Kernel executes the machine level code assigned by the shell with the help of system peripherals. The system peripherals will execute the following commands and return a response to the kernel. The resultant command is sent to shell (which is in machine level language) which converts the response to high level language and presents the response to user on user interface

Level 4- User Interface:

  • User interface is a place where all the applcation software performs it's operations
  • Here the user directly interacts with system via application software (example: MS Word, VLC Media player and other applications) i.e it (system) receives commands from the user via application software(s).
  • Here the user also receives messages from the OS regarding command execution status, errors etc.

Difference between Shell and Kernel:

Shell Kernel
General Definition: Shell is an interface between application software and interface. It also acts as Command line Interpreter. General Definition: Kernel is an interface between hardware peripherals and shell, which executes the commands received from the shell.
The main task of the Shell is: Shell acts as a dictionary of commands. It checks all the commands assigned to the system are valid or not. If the commands assigned to the system are valid, it converts the commands into machine level code and submit it to kernel. The main task of the Kernel is: It receives the command from the Shell and executes them with the help of system peripherals
Shell forms the outer layer of operating system Kernel is the core of the operating system
Shell is responsible for communication between user and system Kernel performs all the tasks assigned by the user and also those tasks which are used to run the system
Shell interacts with user and it interprets the response from Kernel into a language which can be understandable by user Kernel directly interacts with hardware. And Kernel can only understand machine level language
